# Markdown

> **개발 문서** 정리의 기본.

- 텍스트 기반의 마크업 문법

- 태그를 이용하여 문서의 구조를 나타내는 것

- README.md 파일을 통해 오픈 소스의 공식 문서 작성

- 개인 프로젝트의 소개 문서 작성, 학습 내용 정리, 블로그 운영



#### 헤더(#)

> 문서의 제목이나 소제목

*(#)의 개수에 따라 제목의 수준을 구별*



#### 리스트(list)

> 목록을 표시하기 위해 사용

- (1. 2. 3.): 순서가 있는 리스트

- (* -): 순서가 없는 리스트



#### 코드블럭

- (```)	코드를 예쁘게 출력

  ```String str = "hello";```

- ( `)	인라인 코드블럭



#### 링크

- [string]\(링크주소) 링크: string은 보여지는 부분, url은 연결할 곳

- ![string]() : 이미지 삽입



#### 기타

- 텍스트강조
  - **굵게**, *기울임*, ~~취소선~~
  - *대신 _ 사용 가능

- 수평선
  - ---세개 이상
  - *이나 _로 대체 가능

- 인용문(>)
  - (>>) 인용문 안에 하위 인용문

- 테이블 (| 사용, 표 삽입 사용)
